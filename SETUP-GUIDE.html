Everything you need to go live. Hosting, domain, Cal.com, GHL automations, Meta Pixel, and the pre-call form. Follow this start to finish.

3. Deploy to Vercel (Free)

1

Create a GitHub account (if you do not have one)

Go to github.com and sign up. Free.

2

Create a new repository

Click "New Repository" > Name it elevatex-site > Make it Private > Create.

3

Upload your files

Click "Upload files" and drag in the entire elevatex-site folder contents (vercel.json + pages folder). Commit the changes.

4

Connect to Vercel

Go to vercel.com > Sign up with GitHub > "Import Project" > Select your elevatex-site repo > Click Deploy. It will be live in under 60 seconds at a *.vercel.app URL.

5

Add your custom domain

In Vercel dashboard > Your project > Settings > Domains > Add elevatexsolutions.co.uk. Vercel gives you DNS records to add at your domain registrar. Copy the CNAME or A record values and add them in Namecheap DNS settings. SSL is automatic and free.

Making changes: Edit any HTML file on GitHub (click the file > pencil icon > edit > commit). Vercel auto-deploys within seconds. Or clone the repo locally, edit in VS Code, and push. Every push = automatic deployment.

4. Set Up Cal.com

1

Create your event type

Go to cal.com > Event Types > New > Name it "Free Strategy Call" > Set duration to 30 minutes > Add your availability (e.g. Mon to Fri, 9am to 5pm UK time).

2

Configure booking questions

In the event type, add required fields: Name, Email, Phone Number. Add an optional field: "Company Name". These are collected at booking time before they reach the pre-call form.

3

Set redirect after booking

In Event Type > Advanced > Redirect URL after booking, set it to:
https://elevatexsolutions.co.uk/qualify
This sends them straight to your pre-call form after they book.

4

Update the landing page code

In pages/dfy.html, find the Cal.com embed script near the bottom and replace:

var CAL_USERNAME = "YOUR_CAL_USERNAME";   // your cal.com username
var EVENT_SLUG = "strategy-call";         // your event type slug

Your event slug is the URL path shown in Cal.com, e.g. if your booking link is cal.com/alex-elevatex/strategy-call then username is alex-elevatex and slug is strategy-call.

5. Connect Cal.com to GHL

This is how booking data flows into your GHL pipeline automatically.

1

Create GHL webhook

In GHL > Automation > Workflows > Create Workflow > Add Trigger > Select "Inbound Webhook". Copy the webhook URL it generates (looks like https://services.leadconnectorhq.com/hooks/abc123...).

2

Add webhook to Cal.com

In Cal.com > Settings > Developer > Webhooks > Add > Paste your GHL webhook URL > Select trigger: "Booking Created" > Save.

3

Map the data in GHL

In your GHL workflow, after the webhook trigger, add a "Create/Update Contact" action. Map the incoming fields:

First Name  →  {{payload.responses.name.value}}
Email       →  {{payload.responses.email.value}}
Phone       →  {{payload.responses.phone.value}}
Tag         →  "call-booked"
Pipeline    →  "ElevateX Sales"
Stage       →  "Call Booked"

Note: Cal.com webhook payload structure may vary. After adding the webhook, do a test booking and check the GHL workflow logs to see the exact field paths. Adjust mapping accordingly.

6. Connect Pre-Call Form to GHL

1

Create a second GHL webhook (or reuse)

Create another inbound webhook in GHL Workflows for the form submissions. Copy the URL.

2

Update the form code

In pages/qualify.html, find this line near the bottom and paste your webhook URL:

const GHL_WEBHOOK_URL = "https://services.leadconnectorhq.com/hooks/YOUR_WEBHOOK_ID";
3

Create GHL custom fields

In GHL > Settings > Custom Fields > Create these fields (all type "Text"):

industry
monthly_revenue
current_marketing
biggest_challenge
current_tools
marketing_budget
urgency_level
additional_notes

In the workflow, map the incoming webhook data to these custom fields on the contact record.

4

Build the automation workflow

After the contact is created/updated from the form, add these actions:

1. Add Tag: "pre-call-form-completed"
2. Move to Pipeline Stage: "Qualified"
3. Internal Notification to you (email/SMS) with their answers
4. Wait 24hrs before call
5. Send reminder SMS: "Hi {first_name}, looking forward to our call tomorrow at {booking_time}. Come ready to talk growth!"
6. Wait until 1hr before call
7. Send reminder SMS: "Your strategy call with Alex from ElevateX is in 1 hour. Talk soon!"

7. Meta Pixel Setup

1

Get your Pixel code

Meta Business Suite > Events Manager > Data Sources > Your Pixel > Settings > Copy the base pixel code.

2

Add to every page

In each HTML file, find the comment <!-- META PIXEL --> near the top and paste your pixel code there. Then add event tracking:

// On landing page load (dfy.html):
fbq('track', 'ViewContent', {content_name: 'DFY Landing Page'});

// On Cal.com booking complete (add to Cal.com thank you redirect):
fbq('track', 'Schedule', {content_name: 'Strategy Call Booked'});

// On form submission (already in qualify.html, add before webhook call):
fbq('track', 'Lead', {content_name: 'Pre-Call Form Completed'});

8. GHL Pipeline Setup

Create pipeline: "ElevateX Sales"

Go to GHL > Opportunities > Pipelines > Create Pipeline with these stages:

1. New Lead          (Meta ad click, no action yet)
2. Call Booked       (Cal.com booking confirmed)
3. Qualified         (Pre-call form completed)
4. Call Completed    (Manual move after call)
5. Proposal Sent     (Manual move after sending proposal)
6. Negotiation       (If they need time to decide)
7. Closed Won        (Client signed, start onboarding)
8. Closed Lost       (Did not convert, add to nurture)

10. How to Edit Your Pages

Quick edits (text, pricing, copy changes)

Go to GitHub > your repo > click the file > pencil icon > edit directly in the browser > commit changes. Vercel deploys automatically in ~10 seconds.

Bigger changes (design, layout, new sections)

Clone the repo locally with git clone. Open in VS Code. Make changes. Preview by opening the HTML file in your browser. Push to GitHub with git push. Vercel auto deploys. Or ask me to make the changes and I will give you the updated file to upload.

Adding new pages

Create a new HTML file in the pages/ folder. Add a route in vercel.json. Push to GitHub. Done.
